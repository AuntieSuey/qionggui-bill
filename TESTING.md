# 穷鬼账单 - 测试指南

本文档提供详细的测试步骤，确保MVP功能正常工作。

## 前置准备

### 1. 环境检查

```bash
# 检查Node.js版本
node --version  # 应 >= 18.x

# 检查npm版本
npm --version   # 应 >= 9.x

# 检查Expo CLI
npx expo --version
```

### 2. 安装依赖

```bash
cd qionggui-bill
npm install --legacy-peer-deps
```

### 3. 清理缓存（如有问题）

```bash
npx expo start --clear
```

## 自动化测试脚本（可选）

创建一个简单的脚本来批量添加测试数据：

```javascript
// scripts/seed-test-data.js
const { transactionService } = require('../services/database/TransactionService');

async function seedData() {
  const categories = ['food', 'transport', 'shopping', 'entertainment'];
  const types = ['expense', 'income'];

  for (let i = 0; i < 50; i++) {
    const date = new Date();
    date.setDate(date.getDate() - Math.floor(Math.random() * 30));

    await transactionService.create({
      description: `测试账单 ${i + 1}`,
      amount: Math.floor(Math.random() * 10000) + 100,
      category: categories[Math.floor(Math.random() * categories.length)],
      type: types[Math.floor(Math.random() * types.length)],
      date: date.toISOString(),
      source: 'manual',
    });
  }

  console.log('✅ 测试数据添加完成');
}

seedData().catch(console.error);
```

运行：
```bash
npx expo start --clear
# 在另一终端
node scripts/seed-test-data.js
```

## 常见问题排查

### 问题1: 数据库初始化失败
**错误��息**: `SQLite error: no such table: transactions`

**解决**:
1. 删除应用数据重新启动
2. 或检查 `db.ts` 中的 `createTables` 函数

### 问题2: 分类不显示
**原因**: 默认分类未正确插入

**解决**:
1. 检查 `seedDefaultCategories` 函数
2. 确认数据库表 `categories` 结构正确

### 问题3: 智能解析不工作
**原因**: 描述输入时可能未触发解析

**解决**:
1. 确保输入包含金额格式（如"12元"、"¥12"）
2. 检查 `parseInputText` 函数逻辑
3. 查看控制台是否有错误

### 问题4: 统计页面空白
**原因**: `victory-native` 可能未正确配置

**解决**:
1. 当前统计页使用简化图表，避免使用 Victory
2. 检查 `renderBarChart` 函数
